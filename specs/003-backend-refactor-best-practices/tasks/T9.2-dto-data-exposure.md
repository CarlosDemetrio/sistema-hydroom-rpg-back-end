# T9.2 - DTO Data Exposure

## Objective
Ensure DTOs do not expose internal database or audit fields.

## Steps
1. Review response DTOs for audit/internal fields.
2. Remove or mask fields not required by API.
3. Confirm with OWASP guidelines.

## Acceptance Checks
- DTOs only expose necessary public fields.
- Internal IDs/audit fields are excluded unless required.
