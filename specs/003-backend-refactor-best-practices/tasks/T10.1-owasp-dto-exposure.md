# T10.1 - OWASP DTO Exposure Audit

## Objective
Audit DTOs and error responses for sensitive data exposure per OWASP guidance.

## Steps
1. Review response DTOs for sensitive/internal fields.
2. Confirm errors do not leak internal identifiers.
3. Align with `11-owasp-security.md` guidance.

## Acceptance Checks
- DTOs expose only intended fields.
- Error responses avoid sensitive data.
