# T6.4 - Swagger Metadata Audit

## Objective
Ensure that all APIs have the proper Swagger metadata annotations.

## Steps
1. Generate a list of all API endpoints in the project.
2. For each endpoint, check if there is a corresponding Swagger metadata annotation.
3. If a Swagger metadata annotation is missing, add it following the project's guidelines.
4. Verify that the Swagger metadata is correctly configured for each API.

## Acceptance Criteria
- [ ] All API endpoints have the proper Swagger metadata annotations.
- [ ] The Swagger metadata is correctly configured.
