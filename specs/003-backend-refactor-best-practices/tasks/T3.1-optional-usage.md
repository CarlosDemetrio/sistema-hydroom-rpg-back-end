# T3.1 - Optional Usage Standardization

## Objective
Standardize Optional usage across repositories and services.

## Steps
1. Identify Optional-returning methods and current handling.
2. Use `orElseThrow` with domain exceptions consistently.
3. Remove null returns at service boundary.

## Acceptance Checks
- All Optional reads map to domain exceptions on empty.
- No service returns null for missing data.
