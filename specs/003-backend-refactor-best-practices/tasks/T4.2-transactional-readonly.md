# T4.2 - Transactional Boundaries

## Objective
Ensure write operations are transactional and read operations are read-only.

## Steps
1. Review service methods and annotate writes with `@Transactional`.
2. Ensure read methods rely on `@Transactional(readOnly = true)` at class level.
3. Confirm nested transactions are not misconfigured.

## Acceptance Checks
- All writes are transactional.
- Read methods remain read-only by default.
