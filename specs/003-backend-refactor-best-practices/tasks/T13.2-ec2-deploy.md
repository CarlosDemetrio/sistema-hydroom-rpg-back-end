# T13.2 - EC2 Deployment Automation

## Objective
Automate deployment to EC2 (artifact copy + restart) using AWS SSM.

## Steps
1. Deploy artifact to EC2 using SSM or rsync/scp from CI.
2. Restart service via systemd.
3. Validate health endpoint post-deploy.

## Acceptance Checks
- Deployment succeeds with minimal downtime.
- Health check passes after deploy.
