# T2.1 - Service Exception Refactor

## Objective
Replace `IllegalArgumentException` and generic exceptions with domain-specific ones.

## Steps
1. Scan services for `IllegalArgumentException` and generic throws.
2. Map each case to `ResourceNotFoundException`, `ConflictException`, or `BusinessException`.
3. Standardize error messages with `ValidationMessages` when applicable.

## Acceptance Checks
- No `IllegalArgumentException` remains in services.
- Exceptions map to correct HTTP status in handler.
