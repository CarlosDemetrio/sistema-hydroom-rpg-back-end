# T4.1 - Service-Level Validations

## Objective
Add business validations for configuration rules in services.

## Scope
`ConfiguracaoService` and any other services that manage config entities.

## Steps
1. Identify rules: uniqueness, ordering, formula constraints, and soft delete behavior.
2. Implement validations in service layer only.
3. Map violations to `BusinessException` or `ConflictException`.

## Formula Validation Prep
- Centralize allowed variables (ALL CAPS) and functions list.
- Add service-level pre-check for formula fields (syntax + allowed variables), without executing calculations.
- Provide user-facing validation errors via `ValidationMessages`.

## Acceptance Checks
- All rules enforced at service layer.
- Violations return correct exception type.
